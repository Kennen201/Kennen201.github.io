WSUS (windows server update service)

tout d'abord, créer un serveur avec un AD la classique 

ensuite créer un deuxième serveur ou on va installer le service WSUS

Note : sur certaines versions de Windows Server, le rôle s’appelle « Service WSUS ».

À l’étape suivante, cliquez tout simplement sur « Suivant ».

on garde les paramètres par défaut jusqu'à cette page

ici on choisira un dossier partagé de préférence, avec une addresse de partage de type" \\SRV-WSUS\update"

on garde les paramètres par défaut pour le reste, ça va installer IIS

ensuite on ouvre WSUS depuis le gestionnaire de serveur ou depuis windows

ensuite on garde par défaut jusqu'à choisir la synchronisation depuis microsoft update

 

ici on se connecte aux serveur microsoft update ( c long )

on choisit ensuite les langues pour les mises à jours (anglais et français suffiront)

ensuite on va choisir les produits dont on a besoin pour les mises à jours ici en l'occurence Windows 10 et Windows Server 2019

Dans un environnement où il y a la volonté de distribuer les mises à niveau de Windows via WSUS, il sera nécessaire de cocher l’option « Upgrades », et prendre les mises à jours nécéssaires pour windows avec

La synchronisation des mises à jour avec les serveurs Microsoft Update doit être planifiée afin d’être sûr de recevoir les dernières mises à jour. De préférence, cette synchronisation sera planifiée la nuit afin de ne pas perturber la production (gestion de la bande passante). Par exemple, une fois par jour à 02:00 du matin comme sur l’exemple ci-dessous.

Cochez l’option « Commencer la synchronisation initiale » / « Begin initial synchronization » si vous souhaitez réaliser une première synchronisation dès maintenant. (c long)

Cliquez sur « Terminer » / « Finish » : l’initialisation de WSUS est terminée !
