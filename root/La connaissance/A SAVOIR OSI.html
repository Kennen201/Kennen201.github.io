COMMUTATION TABLE DE COMMUTATION

Quels éléments retrouve-t-on dans une table de commutation ?

les adresses mac associées aux ports/interfaces du commutateur

A partir de quel champ de la PDU entrante est-elle bâtie ? 

elle est bâtie à partir de l'adresse mac source de la pdu entrante

Comment un commutateur décide-t-il sur quel(s) port(s) il doit transférer une PDU entrante ?

le commutatueur examine la table de commutaion pour trouver l'adresse mac source. si l'adresse est présente, e commutateur transfère la pdu vers le port associé à cette aresse. si l'adresse n'est pas présente le commutaeur apprend et enregistre l'adresse mac associé au port d'où provient la pdu puis transfère la pdu vers ce port

 

TABLE ARP

 

a quoi ça sert

elle permet de résoudre des adresses ip en adresses mac

on trouve quoi dedans 

table d'adresse ip et adresse mac

comment elle se bâtie 

le switch récupère les adresses mac et ip des machines et lors des demandes il y a une requête arp entre le switch et la machine pour savoir par où doit aller le paquet

 

table de commutation

 

Quels éléments retrouve-t-on dans une table de commutation ?

on retrouve adresse mac et port/interface(g0/1 par exemple)

A partir de quel champ de la PDU entrante est-elle bâtie ? 

en regardant les trames entrante et consultant les adresses mac source et associe les adresse mac aux ports par lequel la trame est rentrée

Comment un commutateur décide-t-il sur quel(s) port(s) il doit transférer une PDU entrante ?

sur une trame unicast : il regarde dans sa table pour voir sur quel interface il doit envoyaer la trame

sur une trame broadcast : le switch transmet la trame sur tout les ports sauf par lequel il est entré

Préfixe par défaut = /8

Nombre d’octet(s) d ‘hôte = 3

Valeurs du 1er octet = 128 à 191

Masque par défaut = 255.255.0.0

Préfixe par défaut = /16

Nombre d’octet(s) de réseau = 2

 Nombre d’octet(s) d‘hôte = 2
