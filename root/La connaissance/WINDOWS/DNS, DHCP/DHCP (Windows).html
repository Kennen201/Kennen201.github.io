DHCP (Windows)

c quoi en fait

Le DHCP attribue automatiquement une adresse IP à une nouvelle machine qui se connecte au réseau si ses préférences sont définies sur obtenir une adresse IP automatiquement (DHCP) au lieu de définir une IP fixe.

C'est pratique car ça évite notamment un grand travail à la main pour les techniciens réseaux.

Port

67 sur DHCPDISCOVER (la machine envoie un message pour découvrir si un DHCP est existant), 68 pour le retour du ou des serveur DHCP, (nommé DHCPOFFER)

67 SERVEUR 68 CLIENT

Acronyme

Dynamic Host Configuration Protocol (Protocole de configuration dynamique des hôtes)

comment ça bosse

machine avec carte réseau sans adresse IP sur un nouveau réseau = envoie d'un message pour savoir si un serveur DHCP peut lui en fournir une
=> DHCPDISCOVER

tout serveur DHCP sur le réseau en mesure de répondre envoie une offre d'adresse à la machine hôte du message
=>DHCPOFFER

la machine hôte envoie une réponse, le serveur stock l'adresse ip dans sa base de donnée, et les autres serveurs qui ne recoivent pas ce paquet s'en servent comme indication pour considérer qu'ils n'ont pas été choisi comme serveur DHCP.
=>DHCPREQUEST

le serveur peut délivrer l'adresse initialement prévu, alors il envoie un paquet contenant l'adresse de la machine à l'hôte et divers paramètres en plus, passerelle par défaut, subnet mask, serveur DNS.
=>DHCPACK

si le serveur ne peut au final pas délivrer l'adresse IP initialement offerte
=>DHCPNAK

si la machine hôte reçoit une ip déjà utilisé 
=>DHCPDECLINE

lorsque le client se libère de son IP/ne renouvelle pas le bail
=>DHCPRELEASE

le client souhaite obtenir des infos sur les paramètres
=>DHCPINFORM
